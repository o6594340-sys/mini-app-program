# Event Mini-App — Стамбул 2026

PWA-приложение для корпоративного инсентив-тура ТехноНИКОЛЬ, 25–28 мая 2026, Стамбул.

---

## Что это такое

Это мини-сайт, который участники тура открывают на телефоне. Он работает как приложение: его можно добавить на экран (как иконку) — тогда запускается без браузера. Работает офлайн после первого открытия.

Два режима:
- **Участник** — открывает сайт, смотрит программу, отель, достопримечательности, контакты
- **Организатор** — заходит в панель администратора, редактирует всё содержимое

## Как работают данные

Вся информация хранится в двух местах:

1. **Файл `data.js`** — данные по умолчанию (Стамбул 2026). Они показываются, если администратор ничего не менял.
2. **localStorage браузера** — изменения из панели администратора. Сайт участника автоматически берёт их, если они есть.

**Важно:** оба файла (`index.html` и `admin.html`) должны быть на одном домене. Иначе участник не увидит изменения организатора.

## Как адаптировать под другое событие

Есть два способа:

**Быстро — через панель:**
1. Открыть `admin.html`
2. Раздел «Данные» → «Шаблон страны» → выбрать нужное направление
3. Отредактировать под конкретный тур

**Полная замена данных — через код:**
1. Открыть файл `app/js/data.js`
2. Отредактировать константы `EVENT`, `HOTEL`, `DAYS`, `SIGHTS`, `CONTACTS` и т.д.
3. Обновить `app/manifest.json` — поле `name` и `short_name`
4. Обновить `<title>` в `app/index.html`
5. **Обязательно поднять версию кэша** в `app/sw.js` (например, `mice-v6` → `mice-v7`) — иначе участники получат старый контент

## Деплой (публикация в интернет)

Это статический сайт — сервер не нужен. Достаточно загрузить папку `app/` на любой хостинг.

**Текущий деплой:** GitHub Pages (бесплатно).

Как обновить после изменений в коде:
```
cd app
git add .
git commit -m "описание изменений"
git push
```
Через ~1 минуту GitHub Pages обновится автоматически.

**Альтернативные хостинги:**
- Beget / TimeWeb / REG.RU — дёшево, поддержка на русском
- GitHub Pages — бесплатно (используем сейчас)

## Куда движется продукт (Roadmap)

Текущая версия — **прототип** для демонстрации и первых продаж.

**Целевая модель:** SaaS-подписка для MICE-агентств из РФ.

| Что | Как |
|-----|-----|
| Каждое агентство — свой аккаунт | Firebase Auth |
| Данные агентства видны всем участникам на их телефонах | Firebase Firestore (облачная БД) |
| Фото — приватная библиотека каждого агентства | Firebase Storage |
| Менеджеры работают с разных компьютеров | синхронизация через облако |
| Тарифы | Стандарт / PRO (разный лимит событий, доступ к общей фотобиблиотеке) |

**Важное ограничение текущей версии:** данные из админки хранятся в `localStorage` — только на том устройстве, где работал администратор. Участники на своих телефонах видят данные по умолчанию из `data.js`. Это нормально для прототипа/демо, но требует переработки на Firebase перед коммерческим запуском.

**Следующий шаг:** когда появятся первые платящие клиенты → переходить на Firebase. Оценка: 2–3 недели разработки.
